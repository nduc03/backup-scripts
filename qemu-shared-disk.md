# 🖇️ Shared Disk Image Between VM and Host (QEMU/KVM + Virt-Manager)

This guide walks you through creating and mounting a shared disk image (`share.img`) that can be accessed both by a Linux host and a Windows guest VM. Great for file transfers, backups, or shared workspace.


## 🏗️ Part 1: Create the Shared Disk Image

Use `qemu-img` to create a raw image file on the host:

```bash
qemu-img create -f raw ~/vm-shared/share.img <size>G
````

> Replace `<size>` with desired size in GB, e.g., `2G`.


## 🖥️ Part 2: Attach Disk Image to Guest VM

1. **Open `virt-manager`**
2. Select your virtual machine → **Open**
3. Click **"Show virtual hardware details"** (top bar)
4. Click **"Add Hardware"** → **Storage**
5. Choose:

   * **Select or create custom storage** → Point to `~/vm-shared/share.img`
   * **Device Type:** Disk
   * **Bus Type:** `VirtIO`
6. Click **Finish**, then **Apply**


## 🪟 Part 3: Setup in Windows Guest

### 1. **Install WinFsp**

* Download and install:
  [https://github.com/winfsp/winfsp/releases](https://github.com/winfsp/winfsp/releases)

### 2. **Install VirtIO Drivers**

* Download and install:
  [https://github.com/virtio-win/virtio-win-pkg-scripts/blob/master/README.md](https://github.com/virtio-win/virtio-win-pkg-scripts/blob/master/README.md)

> ⚠️ **Important:**
> Ensure your VM is using **legacy BIOS** or **disable Secure Boot** for VirtIO drivers to load properly.

### 3. **Format New Disk as FAT32**

Once Windows detects the new disk:

1. Open **Disk Management** (`diskmgmt.msc`)
2. Find the unallocated disk (should match the size of `share.img`)
3. **Initialize the disk** if prompted (use MBR for max compatibility)
4. Right-click the unallocated space → **New Simple Volume**
5. Format the volume using **FAT32** (recommended for cross-platform support)

You can now read/write to this disk from Windows.


## 🐧 Part 4: Mount the Image on the Linux Host

### Step 1: Create Mount Point

```bash
sudo mkdir -p /mnt/vm-shared
```

### Step 2: Attach Image as Loop Device (with Partition Scan)

```bash
sudo losetup --find --show --partscan ~/vm-shared/share.img
```

This returns something like `/dev/loop29`.

### Step 3: Check for Partition Devices

```bash
ls /dev/loop29*
# Expect: /dev/loop29  /dev/loop29p1 ...
```

### Step 4: Mount Your Desired Partition

```bash
sudo mount /dev/loop29p1 /mnt/vm-shared
```

Now the shared folder is accessible at `/mnt/vm-shared`.


## 🧹 Cleanup (When You're Done)

```bash
sudo umount /mnt/vm-shared
sudo losetup -d /dev/loop29
```

Replace `loop29` with the actual device shown earlier.


## 🆘 Troubleshooting

### If No `/dev/loopXp1` Appears

This means your kernel didn’t automatically parse the partitions. You’ll need to install and use `kpartx`:

```bash
sudo apt install kpartx    # or use your distro's package manager
```

Then:

```bash
sudo losetup --find --show ~/vm-shared/share.img
sudo kpartx -av /dev/loopX
sudo mount /dev/mapper/loopXp1 /mnt/vm-shared
```

To unmount:

```bash
sudo umount /mnt/vm-shared
sudo kpartx -dv /dev/loopX
sudo losetup -d /dev/loopX
```

## 📝 Notes

* The disk image can be formatted in Windows (e.g., FAT32 or NTFS) for best compatibility.
* Mounting NTFS on Linux may require `ntfs-3g` (install with `sudo apt install ntfs-3g`).
* FAT32 is recommended for simple read/write sharing between Windows and Linux.
